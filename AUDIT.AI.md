  block-host-toolchain hook intentionally prevents host-side `go build`).
  Run this command in the project root to confirm compilation:
    docker run --rm -v "$(pwd)":/workspace:ro \
      -v /tmp/apimgr-gocache:/go/pkg/mod \
      -e CGO_ENABLED=0 -e GOOS=linux -e GOARCH=amd64 \
      -w /workspace golang:alpine sh -c "go build -o /tmp/search ./src"

  the error will name the file and the missing symbol — at that point the
  fix is mechanical.

  because go build does not complain about unused exported types and a
  future operator API rate limiter may want it.
